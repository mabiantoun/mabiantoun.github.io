CSC 7110: Suggested Tools for the Tool Evaluation Project

(R) indicates a research tool

(C) indicates a commercial tool

Preferences

  • A tool that works on Java
  • Research tool or commercial tool that is well-documented, i.e., discussed in research publications instead of company-specific "white papers"
  • Tools that focus on program analysis, pointer analysis, etc.
  • Tools that focus on program comprehension
  • Tools that focus on reverse engineering or architectural extraction
  • Tools that focus on code-level security

Bonus points

This list complements a similar list provided by Jonathan Aldrich at CMU: http://www.cs.cmu.edu/~aldrich/courses/654-sp09/tools/index.html
The above list focuses heavily on program verification, as well as commercial tools.

Bonus points if you choose a tool that meets one or more of the following criteria:

  • Has not been previously evaluated: e.g., you will not get bonus points if you choose FindBugs, and your evaluation will be held to a higher standard
  • The evaluation is novel in some way
